# Python for Analytical Chemists

**Python for Analytical Chemists is an open notebook library for learning analytical data science with realistic simulated spectroscopy, Raman, NIR, and chemometrics workflows.**

It is built for analytical chemists, lab scientists, grad students, and R&D scientists who work with real measurements but may be new to Python. The focus is not software engineering — it is teachable, stackable code that turns raw instrument data into meaning: importing and cleaning data, smoothing and baseline correction, peak detection, calibration curves, PCA, PLS, and more.

> **All notebooks use simulated data with known ground truth.** Datasets are generated by the bundled [`simulated_data`](simulated_data/README.md) package, which records the *true* peak positions, concentrations, drift rates, and labels alongside every dataset. That means every lesson is fully reproducible, runs offline, ships without data-licensing headaches, and lets you grade your analysis against the real answer — something real lab data can never give you.

## Setup

You need Python 3.9 or newer. From a terminal:

```bash
# 1. Clone the repository
git clone https://github.com/chrisp33/Python_for_Analytical_Chemists.git
cd Python_for_Analytical_Chemists

# 2. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 3. Install the project with the notebook tools
pip install -e ".[notebooks]"

# 4. Launch Jupyter
jupyter lab                       # or: jupyter notebook
```

The editable install (`-e`) puts the `simulated_data` package on your path, so every notebook can `from simulated_data import uvvis, nir, raman` no matter how deep it sits in the folder tree. The `[notebooks]` extra adds `matplotlib`, `jupyter`, and `ipykernel`.

## Curriculum

The lessons are organized into nine tracks that follow the analytical workflow — get data in, handle it numerically, clean the signal, interpret it by technique, model it, and put it to work in the lab. The full lesson index, with status for each notebook, lives in **[`notebooks/README.md`](notebooks/README.md)**. A deeper description of objectives, scientific motivation, and prerequisites for every lesson is in [`docs/curriculum.md`](docs/curriculum.md).

The tracks are: Foundations, Scientific Computing, Signal Processing, Spectroscopy, Mass Spectrometry, Chemometrics, Machine Learning, Sensor Analytics, and Real Laboratory Workflows.

## Contributing

Contributions are welcome. The simplest ways to help: open an issue for a bug, an unclear explanation, or a lesson idea; or open a pull request for a fix. If you change the `simulated_data` package, run the tests first:

```bash
pip install -e ".[dev]"
pytest
```

Please keep the project's spirit in mind — simple, readable, heavily commented code that prioritizes clarity over cleverness, and explains the scientific reason behind each step.
